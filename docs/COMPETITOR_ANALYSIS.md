# 競合調査およびギャップ分析レポート

**対象:** Tepora Project
**日付:** 2024年5月
**比較対象:** Jan.ai (主要競合), LM Studio, GPT4All

## 1. エグゼクティブサマリー

Teporaは「エージェント機能（MCP対応）」や「EM-LLM（エピソード記憶）」といった高度な機能において、競合他社と同等、あるいはそれ以上の先進性を持っています。特にMCPストアの実装は、拡張性の面で大きな強みです。

しかし、**「モデルの発見・導入体験（Model Discovery）」** において、主要競合である Jan.ai や LM Studio に大きく遅れをとっています。競合が「アプリ内での検索・ワンクリックダウンロード」を標準機能としているのに対し、Teporaは「HuggingFaceのRepo ID手入力」を求めており、これが一般ユーザーにとって最大の障壁（弱点）となっています。

## 2. 競合製品の分析

### Jan.ai (主要競合)
*   **特徴:** オープンソース、オフライン重視、クリーンなUI。
*   **強み:**
    *   **Jan Hub:** アプリ内で人気モデルをブラウジングし、推奨設定（量子化ランク等）でワンクリックダウンロード可能。
    *   **RAG UX:** チャット欄へのドラッグ＆ドロップでファイルを読み込み、即座に会話可能（Teporaは専用パネルからの追加が必要）。
*   **弱み:** モバイルアプリの不在（Tepora同様）。

### LM Studio
*   **特徴:** モデル実行・検証に特化したプロプライエタリ・ソフトウェア。
*   **強み:**
    *   **Discovery機能:** HuggingFaceと強力に連携し、検索機能が非常に充実している。マシンスペックに応じた「互換性チェック（Compatibility Check）」が表示される。
*   **弱み:** エージェント機能やツール連携（MCP等）はチャット機能に比べると限定的。

## 3. 機能ギャップ詳細

### 🔴 Critical (早急な対応が必要)

#### 1. モデルハブ / ブラウザ機能の欠如
これがTeporaの最大の弱点です。

*   **Teporaの現状:** ユーザーはブラウザでHuggingFaceを開き、モデルを探し、`Repo ID` と `Filename` をコピーし、Teporaの設定画面にペーストする必要があります。
*   **競合の現状:** アプリ内の「Hub」タブで "Llama 3" と検索し、"Download" ボタンを押すだけです。
*   **影響:** 初心者が「どのモデルを使えばいいかわからない」「導入が面倒」と感じ、離脱する原因になります。

### 🟡 Moderate (改善が望ましい)

#### 2. RAG (ドキュメントチャット) のUX
*   **Teporaの現状:** サイドパネルの「RAG Context」を開き、「Add File」ボタンを押してファイルを選択する手順。
*   **競合の現状:** チャットの入力欄（Input Area）にファイルを直接ドラッグ＆ドロップするだけで認識される。
*   **影響:** 直感的な操作性において、競合の方が「チャットアプリ」として自然な挙動をしています。

### 🟢 Strength (Teporaの強み)

#### 3. MCP (Model Context Protocol) ストア
*   **現状:** `McpStoreModal` が実装されており、GUIからMCPサーバーを検索・インストール・設定可能です。
*   **評価:** これは競合に対しても強力な差別化要因であり、維持・強化すべき点です。

#### 4. EM-LLM (エピソード記憶)
*   **現状:** 単純なベクトル検索だけでなく、学術論文に基づいた記憶システムを実装しています。
*   **評価:** 長期的なパートナーAIとしての体験において優位性があります。

## 4. 推奨アクションプラン

### フェーズ1: 短期的なUX改善 (v3.0 正式版に向けて)
1.  **「おすすめモデル」プリセットの実装**
    *   HuggingFace検索の完全実装は工数がかかりますが、`recommended_models.json` のような静的リストを用意し、`AddModelForm` に「おすすめ（Featured）」タブを追加することは容易です。
    *   例: Llama 3, Gemma, Phi-3 などの主要モデルをワンクリックで入力・ダウンロードできるようにする。

2.  **ドラッグ＆ドロップ対応の強化**
    *   チャット入力欄（InputArea）へのファイルドロップを検知し、自動的にRAGコンテキストに追加する処理を追加する。

### フェーズ2: 本格的な機能拡充
1.  **HuggingFace Search API の統合**
    *   LM Studioのように、アプリ内からHuggingFaceを検索できる機能をバックエンドに追加する。
2.  **互換性チェック機能**
    *   ユーザーのVRAM量を検知し、ダウンロードしようとしているモデルが動作するかどうかを事前に警告する機能。

## 5. 結論
Teporaは「高機能なエージェント」としては非常に完成度が高いですが、「LLMランナー」としての基本的な使いやすさ（特にモデル導入）で競合に劣っています。**「モデルを探して入れる」という最初の一歩を劇的に簡単にする**ことが、最も効果的な改善策です。
