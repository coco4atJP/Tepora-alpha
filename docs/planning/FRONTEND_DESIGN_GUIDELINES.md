# Tepora フロントエンド デザインガイドライン (大規模改修向け)

本ドキュメントは、Teporaの大規模改修に伴うフロントエンドの「デザインシステム・UI/UXレイアウト・コンポーネント設計」の仕様を定義するものです。
技術要件（`FRONTEND_TECHNICAL_REQUIREMENTS.md`）と連携し、技術的負債を一掃するための「厳格なデザインルール」として機能します。

## 1. デザインコンセプト

**「温かみ」「喫茶店・紅茶」「シンプル」「エレガント」「高級感」**

旧来のデザインの温かみを維持しつつ、より洗練されたモダンなアプリケーション体験を提供します。

---

## 2. デザイントークン (Foundation)

Tailwind CSS (v4想定) の theme / CSS変数としてマッピングされるべき基礎トークンです。

### 2.1 タイポグラフィ (Typography)

*   **基本方針**: 「視認性最優先」かつ「シンプルでジオメトリックな可愛さ」
*   **アクセント方針**: 「高級感とエレガントさ（喫茶店のメニューのような品格）」

| 役割 | CSS Variables | フォントスタック例 | 適用箇所 |
| :--- | :--- | :--- | :--- |
| **Sans (本文/UI)** | `--font-sans` | `'Nunito', 'Quicksand', 'Zen Kaku Gothic New', sans-serif` | メッセージ本文、ボタン、ナビゲーション、設定フォーム |
| **Serif (見出し/装飾)** | `--font-serif` | `'Playfair Display', 'Lora', 'Shippori Mincho', serif` | セッションタイトル、AIプロファイル名、特別な見出し |

### 2.2 カラーパレット (Colors)

システムは **Light**, **Dark**, **Tepora** の3つのテーマをサポートし、セマンティック（意味論的）なカラー変数を用いて実装します。

#### テーマ定義マトリクス

| Semantic Variable | 役割 | Light (Milk Tea) | Dark (Roastery) | Tepora (Earl Grey) |
| :--- | :--- | :--- | :--- | :--- |
| `--color-bg` | アプリ全体の背景 | `#FAFAFA` (スノー白) | `#181512` (深い黒茶) | `#F9F6F0` (温かいクリーム) |
| `--color-surface` | カード・入力欄・UI背景 | `#FFFFFF` (ピュア白) | `#241F1A` (ダークモカ) | `#FFFFFF` (ピュア白) |
| `--color-primary` | 主要ボタン・アクティブ要素 | `#3B332D` (チャコール) | `#DA9B64` (アンバー) | `#5C3A21` (ディープ紅茶) |
| `--color-secondary` | アクセント・バッジ | `#D66A44` (テラコッタ) | `#453A31` (カカオ) | `#B85A1C` (アンバー) |
| `--color-text-main` | 本文テキスト | `#1A1614` (黒茶) | `#E8E2DA` (クリーム白) | `#2C1E16` (エスプレッソ) |
| `--color-text-muted` | 補足テキスト・アイコン | `#7E736A` (グレー) | `#9E8E81` (アッシュ茶) | `#8C7A6B` (ミルクティー) |
| `--color-border` | 区切り線・ボーダー | `#E5E0DA` | `#332C25` | `#E8E2DB` |

### 2.3 余白と角丸 (Spacing & Radius)

「柔らかさ（可愛さ）」と「シャープさ（高級感）」のバランスをとります。

*   **Radius (角丸)**:
    *   `rounded-md` (6px): 入力フォーム、小さなボタン等の標準。
    *   `rounded-xl` (12px), `rounded-2xl` (16px), `rounded-[24px]`: カード、チャットバブル、コマンドエリア等の大きな構成要素。丸みを帯びつつも子供っぽくならないジオメトリックなR値。
*   **Spacing (余白)**:
    *   Tailwindのデフォルト（4px単位）に準拠。
    *   **Spacious Luxury (余白的贅沢)**: UI内の「詰まっている感」を完全に排除し、通常のWebアプリケーションの2〜3倍の巨大なホワイトスペース（`p-8`, `p-12`, `gap-12`, `gap-16` 等）を大胆に採用することで、ハイエンドなエディトリアルデザインのような高級感を演出する。

## 3. コアコンポーネント設計仕様

Feature-Sliced Design に則り、`shared/ui` (または `components/`) に配置される基盤コンポーネントのデザインルールです。

### 3.1 Button (ボタン)

*   **バリエーション**:
    *   `Primary`: 背景ベタ塗り (`bg-primary text-white/black`)
    *   `Secondary`: 枠線のみ (`border-secondary text-secondary`)
    *   `Ghost`: 背景・枠線なし、ホバー時のみ背景色付与 (`hover:bg-surface/50`)
*   **インタラクション (UX)**:
    *   ホバー時は `scale-105` などの露骨な拡大は避け、`opacity-90` や微細な `bg` 変更といった**上品で控えめなアニメーション**（`duration-200 ease-out`）を採用する。

### 3.2 Chat Bubble (チャットバブル)

*   **User**: `--color-surface` 背景。シャドウ（`shadow-sm`）をつけ、少し浮かせる。
*   **Assistant**: 背景色なし（透明） または `--color-bg` に溶け込ませる。メッセージ左側にAIのアバターまたはSerif体での名前表示を配置し、エレガントさを表現。
*   **Thought (思考プロセス)**: アコーディオンで折りたたみ可能にし、`--color-text-muted` と少し小さめのフォントサイズで視覚的ノイズを抑える。

### 3.3 TextField / Input (入力フォーム)

*   **状態**:
    *   通常時: `border-border bg-surface`
    *   フォーカス時: 太い青い枠線（ブラウザデフォルト）は無効化し、`ring-1 ring-primary border-primary` へと滑らかに遷移させる。

## 4. 全体レイアウト構成 (Layout & Architecture)

アプリケーションの骨格となる画面レイアウトは以下の通りとします。

### 4.1 アプリケーションシェル (App Shell)

*   **Sidebar (左側 / 右側)**:
    *   ナビゲーションとセッション履歴（左）、RAG/Agentのコンテキスト制御（右）を配置。
    *   不要なボーダーラインを引き算し、代わりに深いシャドウ（`box-shadow`）と広範囲な背景ブラー（`backdrop-filter: blur(40px)`）を用いることで、背景から「重力を無視して浮かんでいる」ような質感を高める。
*   **Main Content (中央)**:
    *   チャットビューと、フルスクリーンの設定ビューを切り替えて表示。
    *   入力バー（Command Area）は画面下部中央に浮遊させ、フォーカス時に強く発光・浮上させることで「入力状態」を心地よくフィードバックする。

### 4.2 レスポンシブ・ウィンドウリサイズ対応

*   Tauriアプリとして、ウィンドウ幅を狭めた際（例: 600px以下）は、**Sidebarをオーバーレイ（ドロワー）または完全に隠す**レスポンシブデザインを実装する。
*   これにより、ユーザーが作業用ウィンドウの横に細長くTeporaを置いた際の実用性を担保する。

### 4.3 画面レイアウト・ワイヤーフレーム（ボタン配置）

シンプルさを極限まで追求しつつ、Tepora特有の「ダイアル型UI」を融合させた **"Floating Minimalist + Radial Menu"** アプローチを採用します。（RaycastやSpotlight, Apple Intelligenceのような没入感とスピード感を意識）

#### A: 全体レイアウト（デュアル・サイドバーと極限のミニマリズム）

画面全体を「チャット・対話」のためだけに広く使い、ノイズを極限まで取り除きます。ナビゲーションや設定は、**画面の左右の端に配置された細いアイコン（ヒントアイコン）にホバーした時だけ引き出される「Drawer / Hidden Sidebar」形式**とします。

*   **メイン入力を主役に**: 画面下部中央に独立した「入出ボックス」が浮遊する形（Glassmorphism）をとります。
*   **履歴は上へ流れる**: 過去のチャットログは、入力ボックスの「奥（上）」に向かって流れていき、上部に行くほどフェードアウト（Opacity減衰やグラデーションマスク）していくという演出を取り入れ、ノイズを減らします。
*   **左サイドバー（セッション履歴）**: 画面左端にある小さなリストアイコンにホバーすると展開します。
*   **右サイドバー（Context Operations）** *(Search/Agentモード時限定)*: 画面右端にある小さなデータベースアイコンにホバーすると展開します。RAGの検索対象やパイプラインの設定を行います。

#### B: Input Area（入力フォーム・ダイヤルUI）の構造

ダイヤル型のモード切替UIは、普段はコンパクトなボタンとして入力欄の横に格納しておき、**マウスオーバー時のみ放射状（Radial）に大きく展開**するギミックを導入します。

RAGなどの操作系周辺UIは右サイドバーに退避させたため、入力欄周りはとことんクリーンに保たれます。

```text
+-------------------------------------------------------------+
| [Hist]                                            [RAG/Ctx] |
| (Hover                                              (Hover  |
|  -> Left Sidebar)                                 -> Right) |
|                                                             |
|           [ User Message (Fade out ↑) ]                   |
|                                                             |
|           [  Agent Message           ]                    |
|                                                             |
|   +-----------------------------------------------------+   |
|   |  (Hover) -> [Radial Menu expands]                   |   |
|   |   (O) <- [Compact Dial Icon]      Type here...  [↑] |   |
|   |                                                     |   |
|   |   +-- [Footer Tools] ---------------------------+   |   |
|   |   | [Thinking: On]  [Attach Image]  [Add Tool]  |   |   |
|   |   +---------------------------------------------+   |   |
|   +-----------------------------------------------------+   |
|                                                             |
+-------------------------------------------------------------+
```

*   **Hover-expandable Radial Menu (ダイヤルUI)**:
    *   **格納状態 (Collapsed)**: 入力欄の左端に、現在アクティブなモード（Chat, Search, Agent等）を示す丸いアイコンだけが表示される。
    *   **展開状態 (Expanded)**: 格納状態のアイコンにホバーすると、ふわっと周囲に円形（放射状）のダイヤルメニューが広がり、他のモードや設定(`Settings`ノブ)が選べるようになる。
*   **右サイドバー: RAG Operations Panel (Context Control)** *(Search/Agentモード時限定)*:
    *   入力欄を圧迫しないよう、RAGやContextの設定は右側の巨大なペインに逃がし、詳細にコントロールできるようにします。
    *   `Agentic Deep Search (Query Generation)`, `Local Context`, `Web Search` のトグルスイッチにより、RAGの挙動をON/OFF可能。
    *   `Active Context DB` (SQLiteの検索対象となるチャンク群やドキュメント) をChipで表示し、追加や削除などの管理が行えます。
*   **メイン入力欄**:
    *   枠線や背景を強調しすぎず、背景のGlassmorphismの強さと、フォーカス時の`--color-gold`系のGlow（輝き）で「入力中」であることを示す。
*   **Footer Tools (付加機能群)**:
    *   ツールの追加、画像添付、ThinkingモードのOn/Offなどは、入力欄の下に小さなテキストとアイコン(`text-xs text-muted`)で整列配置。

#### C: Full-Page Settings View (設定画面: フルスクリーン・ウルトラミニマル)

以前の「モーダル（重なり合う箱）」という概念を完全に捨て去り、設定画面を開くと**チャット画面全体がシームレスに設定ドキュメントへと遷移するフルページレイアウト**を採用します。

*   **Editorial & Zen (エディトリアルと禅の精神)**:
    *   枠（Box）や背景色の区切りを極力排除し、**巨大な余白（`padding: 8vh 12vw`）**と**美しいタイポグラフィの強弱のみ**で構造をピュアに表現します。まるで高級ホテルのパンフレットや、ハイエンドな雑誌を読んでいるかのような「余白的贅沢（Spacious Luxury）」を感じさせるデザインです。
*   **構造**:
    *   **左側 (Navigation)**: 非常に大きなセリフ体（Plairfair Display等）を用いたナビゲーションリンク。不要なアイコンは削ぎ落とし、文字そのものの美しさで魅せます。
    *   **右側 (Content Flow)**: 設定項目が縦にゆったりと流れるように配置されます。それぞれの設定グループ間のマージンは通常の2〜3倍（`gap: 64px`）とり、息苦しさを徹底的に排除します。
*   **シームレスな遷移**:
    *   ダイヤルUIの`Settings`ノブをクリックすると、入力欄（コマンドエリア）がスッと下に消え、チャット履歴がフェードアウトするのと入れ替わりに、設定画面が浮かび上がります。
*   **Quiet Controls (静かなコントロール部品)**:
    *   選択肢（テーマやフォント等）は、囲み枠のないプレーンなテキストと、選択状態を示す極小のドットインジケーターのみで表現します。
    *   スライダーも極細の1pxラインに乗った小さなノブという、極限まで要素を削ったミニマルな形状にします。

#### D: Agent Execution Panel (Agentモード専用 右サイドバー)

Agentモード選択時のみ、右サイドバーに展開されるエージェントの行動制御用UIです。Teporaのバックエンドアーキテクチャ（階層的マルチエージェント）を、ユーザーが直感的に操作できる形に落とし込みます。

*   **Target Agent / Skill**:
    *   どのエージェント・スキル（`Coder`, `Writer`, `Supervisor` による自動ルーティング等）を使用するかを選択するドロップダウン。
*   **Execution Mode (実行モード)**:
    *   `High (Planner Loop)`: 事前に計画を立てて長期的・複雑なタスクをこなすモード。
    *   `Low (Direct Execution)`: プロンプトに対して即座に行動を起こす、高速なシングルショット実行モード。
*   **Tools Access (ツール権限設定)**:
    *   そのタスクでファイル編集やターミナル実行など、どのツールへのアクセスを許可するかを可視化（オン・オフ）する領域。



### 4.4 メッセージ操作アクション（各チャットバブル）

メッセージに対するアクション（コピー、再生成、削除など）は、**ホバー時のみ出現**させることで普段のチャット画面のエレガントさを保ちます。

*   **配置**: メッセージバブルの下部、またはバブルの右端付近。
*   **アイコン**:
    *   [Copy] (クリップボード)
    *   [Regenerate] (Userメッセージ横: 鉛筆アイコンで編集・再送信。Assistantメッセージ下: リロードアイコンで別案生成)
    *   [Delete] (ゴミ箱)

## 5. 技術導入プランへの連携 (実装フェーズへの引き継ぎ)

本デザインガイドラインをコードベースに適用する際は、以下の技術要件と統合して進めること。

1.  **Tailwind 設定の再構築**: 古い独自クラスを破棄し、上記の CSS Variables ベースの `theme` 拡張に移行する。
2.  **Lint / 規約の徹底**: プレーンなCSSの直書きを禁止し、全てデザイントークン経由（`text-primary`, `bg-surface` など）でスタイリングを行う。
3.  **コンポーネント化**: `shared/ui` に Button, Input, Select 等の Primitive コンポーネントを作成し、そこでのみ Tailwnind クラスを許容。Feature 側では `className` による上書きを極力行わない。
