# V2 設定UI 項目整理 — できることまとめ

> [!NOTE]
> バックエンド設定インベントリ + V1フロントエンドの知見 + [UI_PROTOTYPE_REFERENCE.html](file:///Users/nekon/dev/Tepora-alpha/docs/planning/UI_PROTOTYPE_REFERENCE.html) を元に、V2設定画面を**再設計**したものです。V1の優れたUX要素は概念レベルで継承しつつ、V2のデザイン言語で一から構築します。

## 設定画面の全体レイアウト — 3段構成

UI Prototypeの"Editorial & Zen"思想をベースに、**上下3段** に再構成します。

```
┌──────────────────────────────────────────────────────────────────────┐
│ ← Return to Session                                                 │
│                                                                      │
│ ① Top Nav Row （ナビゲーション）                                      │
│ ┌──────────────────────────────────────────────────────────────────┐ │
│ │  General  ·  Appearance  ·  Characters  ·  Models  ·  Privacy   │ │
│ │  Tools  ·  Memory  ·  Context  ·  Data  ·  System  ·  Advanced  │ │
│ └──────────────────────────────────────────────────────────────────┘ │
│                                                                      │
│ ② Section Tabs （セクション切り替え）                                 │
│ ┌──────────────────────────────────────────────────────────────────┐ │
│ │  セリフ体の大きなタブ名  |  該当サブセクションが横に並ぶ          │ │
│ │  例: Models選択時 → [Hub] [Defaults] [Embedding] [Loader] [Adv] │ │
│ └──────────────────────────────────────────────────────────────────┘ │
│                                                                      │
│ ③ Content Area （設定コンテンツ） ← スクロール無し・1画面完結         │
│ ┌──────────────────────────────────────────────────────────────────┐ │
│ │                                                                  │ │
│ │  Section Tab切り替えで内容が差し替わる（ページ遷移感覚）         │ │
│ │  各タブのコンテンツはビューポート内に収まる設計                  │ │
│ │                                                                  │ │
│ │  ・selection-dot による選択肢                                    │ │
│ │  ・line-slider による極細スライダー                              │ │
│ │  ・min-toggle による極小トグル                                   │ │
│ │                                                                  │ │
│ └──────────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────────┘
```

### レイヤーの役割

| レイヤー | 役割 | デザイン |
|:---|:---|:---|
| **① Top Nav** | カテゴリ切り替え | ゴールド系のミニマルテキストリンク、アクティブ時にドットインジケーター |
| **② Section Tabs** | サブセクション切り替え | セリフ体の大きなタブ名（Prototypeの `huge-nav-link` スタイル踏襲） |
| **③ Content Area** | 実際の設定項目 | スクロール無し。Section Tab切り替えで内容が差し替わり、各タブは1画面に収まる。巨大余白 + `config-row` + selection-dot / line-slider / min-toggle |

### 遷移アニメーション

- ダイヤルUIの **Settings ノブ** クリック → コマンドエリアがフェードダウン → 設定画面がフェードアップ
- Top Nav 切り替え時は Section Tabs と Content Area がクロスフェード
- **Return to Session** ボタンで逆アニメーション → チャット画面に戻る

### 4. Models 🔧

> [!IMPORTANT]
> V1の「モデルハブ（オーバーレイ）」「ダイアル型パラメータ調整」「多段ロールマッピング」のUXコンセプトをV2デザイン言語で再設計します。

| Section Tab | 内容 |
|:---|:---|
| **Hub** | 「モデルハブを開く」ボタン → **フルスクリーンオーバーレイ**（V1のUXを継承し再設計） |
| **Defaults** | デフォルトモデル割り当て（下表参照） |
| **Embedding** | 埋め込みモデル選択 + 基本パラメータ |
| **Loader** | Ollama / LM Studio の Base URL |
| **Advanced** | グローバルLLMデフォルト、キャッシュサイズ、リランキング、Tokenizer設定 |

#### モデルハブ オーバーレイ（V1のUXを継承し再設計）

V1のカードグリッド＋検索フィルター＋プロバイダースキャンというUXコンセプトを、V2のデザイン言語（glass-bg, selection-dot, 巨大余白）で再構築する。

- **検索 + フィルター**（All / Text / Embedding） — V2の `selection-dot` スタイルで表現
- **プロバイダースキャン** — Ollama / LM Studio からモデル一覧を再取得
- **モデルカード** — クリックでパラメータ調整パネルを開く

#### ダイアル型パラメータ調整（V1のUXを継承し再設計）

V1のダイアル型UIの「直感的にパラメータを調整できる」体験を継承し、V2のミニマル意匠で再設計する。

- **基本パラメータ**（上部）: パス、ポート、コンテキストサイズ、GPUレイヤー
- **サンプリングダイアル**（中央）: V2デザイン言語を適用した放射状ダイアルで Temperature / Top-P / Top-K を調整
- **詳細パラメータ**（折りたたみ）: ペナルティ群、Seed、Mirostat、停止シーケンスなど

#### デフォルトモデル割り当て（V1のロールマッピング概念を継承し再設計）

| 設定項目 | UIコントロール | 説明 |
|:---|:---|:---|
| **キャラクターモデル（グローバル）** | ModelSelector | 全キャラクター共通のデフォルトモデル |
| **キャラクター別モデル** | 各キャラ行にModelSelector | キャラごとに上書き可能（空=グローバル継承） |
| **エージェントスキル別モデル** | 各スキル行にModelSelector | スキルごとに上書き可能 |
| **プロフェッショナルデフォルト** | ModelSelector | 内部システムタスク用のデフォルトモデル |
| **Supervisor/Planner デフォルト** | テキスト入力 | プランナー用モデル名 |
| **Executor デフォルト** | テキスト入力 | エグゼキューター用モデル名 |

## 設計の要点まとめ

| 要素 | V1のエッセンス | V2での再設計方針 |
|:---|:---|:---|
| **設定レイアウト** | 左サイドバー型 | **上下3段**（Nav → Section Tabs → Content）に再構成 |
| **モデルハブ** | カードグリッド+検索+オーバーレイ | UXコンセプトを継承し、V2デザイン言語で**新規構築** |
| **パラメータ調整** | ダイアル型UI | 放射状ダイアルの直感性を継承し、V2意匠に合わせて**再設計** |
| **デフォルトモデル** | 多段ロールマッピング | グローバル→キャラ→スキル→プロフェッショナルの階層を**概念継承** |
| **設定コントロール** | 各種フォーム部品 | Prototypeの **selection-dot / line-slider / min-toggle** に統一 |
| **遷移** | ダイアルUI → フェードイン/アウト → フルページ設定 |
| **デザイン思想** | "Spacious Luxury" — 巨大余白・セリフ体・枠なし・ドットインジケーター |

## 設定UIで「できること」サマリー

1. **言語・キャラクター切り替え** — ワンクリック変更
2. **テーマ・フォント・コードブロックのカスタマイズ** — 4テーマ + フォント調整 **[FE]**
3. **通知とショートカット設定** **[FE]**
4. **キャラクターの完全カスタマイズ** — アバター・プロンプト編集・新規作成
5. **カスタムエージェントの構築** — タグベースルーティング・ツール制限
6. **モデルハブによる一元管理** — 検索/フィルター付きカードグリッドで全モデルを操作
7. **ダイアル型UIでのパラメータ調整** — Temperature/Top-P/Top-Kをダイアル操作
8. **デフォルトモデルの多段マッピング** — グローバル→キャラ→スキル→プロフェッショナルの階層割り当て
9. **プライバシーの厳格な制御** — 隔離モード、URLブロック、ロックダウン
10. **ツール権限と検索プロバイダー管理**
11. **エピソード記憶の制御** — 減衰パラメータまで詳細調整
12. **RAGとコンテキスト配分の最適化**
13. **データ管理とストレージ設定** **[FE]**
14. **暗号化バックアップの作成と復元** **[FE]**
15. **システム設定** — 自動起動、GPU/メモリ制限、アプリ＆エンジン更新 **[FE]**
16. **上級者向けチューニング** — タイムアウト、DLポリシー、フィーチャーフラグ等
