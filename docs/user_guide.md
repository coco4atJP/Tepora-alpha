# Tepora ユーザーガイド

Tepora は、記憶機能とエージェント実行を備えた local-first のデスクトップ AI アプリです。このガイドでは、初回セットアップから日常的な使い方までを現行実装に合わせて整理します。

## 1. Tepora でできること

- **Chat / Search / Agent** の 3 モードで会話と作業支援を切り替えられます。
- **EM-LLM 記憶** により、会話から抽出した長期記憶を保持できます。
- **セッション履歴** から過去の会話を切り替えて再利用できます。
- **モデル管理** で `llama.cpp`、`Ollama`、`LM Studio` を使い分けられます。
- **MCP 連携** により、外部ツールサーバーを追加できます。

> Tepora は local-first ですが、完全オフライン専用ではありません。Web 検索、モデルダウンロード、外部 MCP、OpenAI-compatible 接続は設定と許可がある場合にのみ利用されます。

## 2. インストールと初回起動

### 推奨環境

- Windows 10 / 11 (64-bit)
- 16GB 以上のメモリ推奨
- モデル格納用に十分な空き容量

### 初回セットアップ

初回起動時はセットアップウィザードが表示されます。現在のウィザードは次の順で進みます。

1. **言語選択**
2. **LLM ローダー選択**
   - `llama.cpp (Bundled)`
   - `Ollama`
   - `LM Studio`
3. **要件確認**
4. **モデル設定 / ダウンロード**
5. **インストール完了**

`llama.cpp` は同梱バイナリを使うローカル実行向けです。`Ollama` と `LM Studio` は既存の外部ランタイムに接続します。

## 3. メイン画面

### チャット画面

メイン画面では次の操作ができます。

- 新しいセッションの作成
- セッション履歴の表示
- メッセージ送信とストリーミング応答の確認
- 添付ファイルの送信
- ツール実行時の承認

### モード切替

- **Chat**: 通常の会話
- **Search**: 検索結果や RAG コンテキストを併用した回答
- **Agent**: ツール呼び出しや進捗表示を伴う実行

### 画面メニュー

- **Logs**: バックエンド / フロントエンドのログ閲覧
- **Memory**: 記憶統計と compaction job の確認、圧縮実行
- **Settings**: 全般、プライバシー、データ保存、性能、エージェント、Agent Skills、MCP、モデル、メモリ設定

## 4. 記憶とセッション

### セッション

- 会話はセッション単位で保存されます。
- サイドバーや履歴ダイアログから、既存セッションへの切り替えや削除が可能です。

### 記憶

- Tepora は会話から EM-LLM 記憶を生成します。
- `Memory` ページでは、Character / Professional メモリの統計を確認できます。
- `Compress Memories` を実行すると、記憶圧縮ジョブがバックグラウンドで走り、履歴が一覧表示されます。

## 5. 設定の要点

### モデル

- 既定モデルの切り替え
- Character / Professional / Embedding 用の割り当て
- Ollama / LM Studio のモデル再スキャン
- ローカルモデルの登録や削除

### プライバシー

- `allow_web_search` で Web 検索を制御
- `lockdown` で外部アクセスや危険操作をより強く制限
- URL ポリシーと権限 TTL の調整

### Agent Skills / MCP

- Agent Skills は UI から一覧・保存・削除が可能です。
- MCP サーバーは設定、承認、無効化、削除を行えます。

## 6. よくある確認ポイント

**Q. 起動直後に接続エラーになります。**
A. バックエンド sidecar の起動待ちで数秒かかることがあります。改善しない場合は `Logs` 画面でエラー内容を確認してください。

**Q. Web 検索が動きません。**
A. `privacy.allow_web_search` が無効、または Lockdown が有効だと検索はブロックされます。

**Q. Ollama / LM Studio のモデルが出てきません。**
A. 各ランタイムが起動していることを確認し、モデル画面から再スキャンを実行してください。

**Q. オフラインで使えますか。**
A. `llama.cpp` とローカルモデルを使う範囲ではオフライン運用が可能です。モデルダウンロードや Web 検索はオンライン接続が必要です。
